# 森のカフェ ランディングページ — 社内プレゼン資料

**ドキュメント種別**: プロジェクト概要（背景・ターゲット・施策・成果目標）  
**対象**: 社内向け  
**最終更新**: 2026年4月（プロジェクト現状に基づく）

---

## 1. プロジェクトの背景

### 1.1 目的

カフェブランド「**森のカフェ**」を題材にした、**単一 HTML** のランディングページ（LP）です。フレームワークに依存しない**静的サイト**として、マークアップとスタイルの設計・訴求構成を一枚の `index.html` に集約しています。

### 1.2 位置づけ

- **デモ案件**: 店名・住所・営業情報は**架空**であり、実在店舗を前提とした運用データではありません。
- **技術・表現の検証**: ファミリー向け訴求と焙煎・抽出へのこだわりを**同一ページ内で両立**させる構成・トーンの試作として位置づけられています。
- **運用入口**: ワークスペース直下の [CAFE.md](../CAFE.md) から本プロジェクトの README へ誘導する構成です。

### 1.3 解決したい課題（デモ上の前提）

- 小規模カフェ向けに、**重いビルド環境なし**で更新・プレビューできる資産にすること。
- **家族連れ**と**コーヒーへのこだわり**の両方を、離散させずに伝える情報設計の整理。

## 2. ターゲット

### 2.1 主ターゲット（サイト上の訴求）

| 観点 | 内容 |
|------|------|
| **コア** | **ファミリー層**（お子さま連れ歓迎を明示） |
| **ニーズ** | 休日のカフェ時間、公園帰り・週末ブランチなど「ちいさな冒険のあと」の立ち寄り |
| **提供価値** | キッズドリンク・シェア向きスイーツ・甘さ控えめメニュー、ママ・パパ向けデカフェなど、**世代横断で選べるメニュー** |

### 2.2 サブターゲット（ストーリー・メニュー周り）

- **コーヒーに関心のある大人**: 産地の個性を活かす焙煎、ハンドドリップ・ネル・フレンチプレスのみ（エスプレッソ非掲載）など、**抽出と豆へのこだわり**を説明するセクションで接続。

### 2.3 トーン・メッセージ

- **ヒーロー周辺**: カジュアルな文体（ファミリー・休日の文脈）。
- **ことば（ストーリー）**: 焙煎・ドリップに寄せた、やや専門的なトーン。
- **キャッチ（例）**: 「休日のカフェ時間、ちいさな冒険のあとに。」（基本情報と同期）

## 3. 施策

### 3.1 情報設計（ページ構成）

1. **固定ナビ** — ロゴ、ことば / お品書き / 店舗へのアンカー  
2. **ヒーロー** — ファミリー向けキャッチ、リード、来店・メニューへの CTA  
3. **ことば（#story）** — 焙煎と抽出（ハンドドリップ・ネル・プレス）の紹介  
4. **お品書き（#menu）** — ダーク背景の 3 カラム（ママ・パパの一杯 / キッズ＆ジュニア / みんなでシェア）  
5. **店舗（#visit）** — 営業時間・所在地（デモ用プレースホルダ）  
6. **フッター** — ブランド名・デモ表記  

### 3.2 デザイン・表現

- **配色**: クリーム／紙色ベース、テラコッタ系アクセント、深緑のヒーロー視覚エリア  
- **フォント**: Google Fonts — Cormorant Garamond（ディスプレイ）、Zen Kaku Gothic New（本文）  
- **演出**: 軽いフィルムノイズ風オーバーレイ、ヒーローのフェードイン、ホバー時のカード浮き  

### 3.3 技術施策

- **HTML5**、スタイルは **同一ファイル内の埋め込み CSS**（外部 JS なし）  
- フォントは **CDN**（`fonts.googleapis.com`）読み込み  
- **プレビュー**: `npm run preview:cafe`（Python 3 `http.server`、既定 `http://127.0.0.1:5173/`）、または `npm run preview:cafe:serve`（`npx serve`）。VS Code / Cursor のタスク・デバッグ構成からの起動にも対応  

### 3.4 コンテンツ・データ整備

- 店名・営業・メニュー等の一覧を **[基本情報.md](基本情報.md)** に整理し、**[基本情報.yaml](基本情報.yaml)** / **[基本情報.json](基本情報.json)** で機械可読化（HTML と同期運用を想定）。

## 4. 成果目標

### 4.1 デモプロジェクトとしての到達目標（現状のゴールイメージ）

| 観点 | 目標 |
|------|------|
| **資産** | 単一 HTML で LP を完結させ、依存関係とデプロイ対象を最小化する |
| **一貫性** | README のメタ表・基本情報・`index.html` の文言・構造を揃え、**更新手順が明文化**されている |
| **再現性** | ローカル・エディタから **短いコマンドまたはタスク** でプレビューできる |

### 4.2 本番化・拡張時に想定する成果指標（方向性）

README の「今後の拡張例」をベースに、実店舗や本番運用に載せ替える場合の**目安**として整理しています。

- **情報の正確性**: 住所・電話・地図の差し替え、OGP メタタグの整備  
- **訴求の強化**: 写真アセットの追加  
- **スケール**: 必要に応じた別ページ化やビルドツール導入  

（数値 KPI はデモ段階では未定義。本番化時は「予約・来店導線」「問い合わせ」「SNS 遷移」などビジネス要件に合わせて設定することを推奨。）

### 4.3 社内共有で使える一言サマリ

> **架空店舗を題材に、ファミリー向け訴求とコーヒーへのこだわりを一枚の静的 LP で表現。ビルドレス寄りの運用と、基本情報の二重管理（人間可読 + 機械可読）まで含めたデモ資産。**
